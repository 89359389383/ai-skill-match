// このファイルは「モデル（Eloquent）」に関する実装メモです。
// 何をしたかを、やさしい日本語でまとめます。
//
// ■ 目的
// スキル購入後の取引（チャット・納品・承認・評価）を、DBとコードの両方で扱えるようにするために、
// モデルとリレーション（関連）を追加します。
//
// ■ 追加・修正したモデル
//
// 1) SkillOrderMessage（新規）
// - 取引（skill_orders）にぶら下がる「チャットメッセージ」を保存するモデルです。
// - message_type で text / file / system を区別します。
// - system メッセージは「誰が送ったか」が無いので sender_user_id は null になります。
//
// 2) SkillOrder（既存の拡張）
// - messages() リレーションを追加しました。
//   これにより、$order->messages で取引チャットの履歴を取り出せます。
// - fillable / casts に transaction_status / delivered_at / completed_at を追加しました。
//
// 3) User（既存の拡張）
// - skillOrders(): 購入者としての取引（注文）一覧を取得できます。
// - skillOrderMessages(): 取引チャットで「自分が送信したメッセージ」一覧を取得できます。
//
// ■ できるようになったこと（例）
// - 取引ページ表示時に「取引情報 + チャット履歴」をまとめて取得できる
// - system メッセージ（納品・完了など）をDBに残して、UIで通知表示できる
