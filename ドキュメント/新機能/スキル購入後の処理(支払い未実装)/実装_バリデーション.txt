// このファイルは「入力チェック（バリデーション）」に関する実装メモです。
// 何をチェックしているのかを、やさしい日本語でまとめます。
//
// ■ 目的
// 取引チャットや評価はユーザー入力があるため、
// 変な値（空、長すぎる文字、範囲外の評価など）が入らないようにします。
//
// ■ 追加した FormRequest
//
// 1) SkillTransactionMessageRequest
// - 対象: 取引チャットのメッセージ投稿
// - チェック内容:
//   - content は必須
//   - 文字列であること
//   - 2000文字以内
// - 補足:
//   既存との互換のため、body で送られてきた場合は content に読み替えます。
//
// 2) SkillTransactionCompleteRequest
// - 対象: 購入者の「承認＋評価」
// - チェック内容:
//   - rating は必須（1〜5の整数）
//   - review は任意（文字列、2000文字以内）
//
// ■ 認可（誰ができるか）の考え方
// - FormRequest は「入力の形」をチェックします。
// - 「購入者だけが承認できる」「出品者だけが納品できる」などの権限チェックは、
//   これまで通り Controller 側で当事者チェックとして行っています。
