本機能で実装した内容（スキル/質問の導線・編集/削除）

## 1. スキル機能（一覧・詳細・編集・削除）

### 1.1 フリーランスプロフィール詳細から「そのフリーランスのスキル一覧」へ遷移
- `profiles/show`（フリーランス詳細プロフィール画面）の「提供サービス」セクションに「スキル一覧」リンクを追加しました。
- リンク先は `profiles/{user}/skills` で、そのフリーランスが出品しているスキル（公開中）だけが一覧表示されます。

### 1.2 スキル一覧画面 → スキル詳細画面
- `skills/index` では、各スキルカードをクリックすると `skills/show`（スキル詳細）へ遷移します（既存挙動を維持）。

### 1.3 スキル一覧/詳細で本人なら「編集」「削除」を表示
- ログインしているフリーランスが出品者本人の場合に限り、次を表示するようにしました。
  - スキル一覧カードに「編集」「削除」ボタン
  - スキル詳細の右側（購入/問い合わせ領域の代替）に「編集」「削除」ボタン

### 1.4 削除はモーダル確認 → 削除実行
- 「削除」ボタン押下時に削除確認モーダルを表示します。
- モーダル内の「キャンセル」はモーダルを閉じます。
- モーダル内の「削除する」を押すとDELETEで削除が完了します。

### 1.5 編集画面（作成と同等のUI）と更新/キャンセル
- スキル編集画面を追加し、作成画面と同様の項目・見た目で編集できるようにしました。
- 編集画面には「更新」「キャンセル」を設置しています。
- 更新/削除の完了後は「スキル一覧（そのフリーランスのスキル一覧）」へ戻る動作にしています。

### 1.6 タグ/サムネイルの保存整合
- スキルフォームで扱うタグ入力を、バックエンド側では `skill_names[]` として受け取り、`skills` テーブルへ自動登録/紐付けできるようにしました。
- スキルフォームのサムネイルは `thumbnail` アップロードとして受け取り、保存後に `thumbnail_url` へ変換して表示されるようにしました。
- 編集時は `remove_thumbnail`（削除フラグ）でサムネイルを消せるようにしました。
- 作成/編集いずれも同じ入力項目から同じ形で保存されるため、UIと保存内容のズレが解消されます。

### 1.7 スキル詳細画面の「スキル一覧」導線
- `skills/show` の右側エリアに「スキル一覧」リンクを設置し、一覧へ戻れる導線を明確にしました。

## 2. 質問機能（質問一覧・自分の質問一覧・詳細・削除）

### 2.1 質問一覧画面に「自分の質問一覧」ボタン
- `questions/index` に「自分の質問一覧」ボタンを追加しました。
- ボタンは「質問を投稿」ボタンの左側に配置しています。
- ログインしている場合のみ表示されます。

### 2.2 自分の質問一覧（一覧 → 詳細）
- `my-questions` 画面を追加し、ログインユーザーが作成した質問のみを一覧表示します。
- 自分の質問一覧の各質問カードをクリックすると `questions/show`（質問詳細）へ遷移します。

### 2.3 自分の質問一覧/質問詳細で本人なら「削除」
- 自分の質問一覧では、作成者本人の質問にだけ「削除」ボタンを表示します。
- 質問詳細画面でも、作成者本人なら「削除」ボタンを表示します。

### 2.4 削除はモーダル確認 → 削除実行
- 削除はモーダルで確認してから実行します。
- キャンセルはモーダルを閉じます。
- 削除完了後は「自分の質問一覧」へ戻る動作にしています。
